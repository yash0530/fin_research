# docs/ — documentation

- `dev_guide.md` — architecture, invariants, module tour, how to extend (add a tool, an
  agent, a provider), testing strategy, verification. Read this before changing code.
- `user/` — end-user guide (getting started, the daily workflow, dossiers, the monthly
  buy-list ritual, calibration, the capture channel, FAQ).

Docs are prose (not compiled) but are part of the "definitive source of truth": keep them
in sync with behaviour, and update `../TASKS.md` when status changes.
