# v2-5 — 13F superinvestor holdings ingestion + cloning overlap

Context: 13F-HR filings are filed by INSTITUTIONS (fund managers), not by the issuers in our universe — so they are NOT in `EdgarFiling` (which only indexes issuer forms). This batch adds a NEW ingestion path for a small curated set of superinvestor CIKs, parses their holdings, and screens our universe for overlap ("who owns this / is it a cloned superinvestor pick").

Read first: `src/net/edgar.ts` (EDGAR_LIMITER 8 req/s, requireUserAgent, submissions fetch pattern), `src/net/edgar-facts.ts` (companyfacts fetch style + defensive parse), `src/net/edgar-form4.ts` (XML parse via fast-xml-parser, injectable fetch, never-throw), `src/net/CLAUDE.md`, `src/jobs/registry-live.ts` (job wiring, per-item catch), `prisma/migrations/CLAUDE.md` (additive migration style). Migration number: **0011** (0007-0010 exist).

## Build

1. `src/config/superinvestors.ts` — a curated list `SUPERINVESTORS: {cik, name, slug}[]` (Berkshire Hathaway 0001067983, plus ~8 well-known value managers — pick recognizable CIKs; document each). Export helpers.
2. `src/net/edgar-13f.ts` — `fetch13FLatest(cik, fetchImpl, ua, limiter)`: via the submissions API (`data.sec.gov/submissions/CIK##########.json`) find the most recent form `13F-HR`, then fetch its **information table XML** (the `*.xml` in the filing whose root is `informationTable`; resolve the raw doc, stripping any xsl viewer prefix like edgar-form4's `rawForm4Doc`). Parse rows: `{nameOfIssuer, cusip, value /*USD thousands*/, sshPrnamt /*shares*/}`. Defensive: malformed → [] + warning, never throw. Injectable fetch for tests.
3. Migration `0011_holdings.sql` + schema: `InstitutionalHolding(id PK, filerCik TEXT, filerName TEXT, periodOfReport TEXT, cusip TEXT, nameOfIssuer TEXT, value REAL, shares REAL, filedAt TEXT)` + index (filerCik, periodOfReport) and (cusip). Additive only.
4. `src/screens/superinvestor-overlap.ts` — pure: given `InstitutionalHolding[]` (across filers) mapped to our symbols (by CUSIP→symbol lookup passed in), return per-symbol `{symbol, holders: {name, shares, value}[], holderCount, newThisQuarter?: boolean}`. A name held by ≥1 tracked superinvestor gets a `superinvestor` signal; ≥2 = stronger.
5. Job `holdings_13f` in `src/jobs/registry-live.ts`: for each SUPERINVESTOR cik, fetch+parse latest 13F, upsert `InstitutionalHolding` (INSERT OR IGNORE keyed on filerCik+cusip+periodOfReport). Then map CUSIP→symbol using `Ticker` (add a `cusip` column? NO — keep additive: match on `nameOfIssuer`≈`Ticker.name` fuzzy OR skip unmatched, honest partial) and merge a `superinvestor` trigger tag into matched `Candidate`s (never overwrite userState; per-item catch).
