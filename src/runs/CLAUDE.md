# src/runs/ — on-demand research runs

- `budget.ts` — `getBudgetConfig(runType, budgetSeconds)`: maps run types (`ticker_dossier`, `theme_sweep`, `watchlist_reunderwrite`, `filing_diff`, `open_question`, `theme_proposal`) and budgeted seconds to multi-agent configurations and model profile, clamped to 15m–5h.
- `safety.ts` — `checkHardwareThrottling(db, runId, opts)`: checks macOS thermal warnings (`CPU_Speed_Limit < 50` → 30s pause) and battery status (unplugged and `< 25%` → abort run) to protect hardware during model inference.
- `reconcile.ts` — `reconcileRuns(db, opts)`: reclaims database states of orphaned runs (where the process PID has died) to `FAILED` and releases stale cross-process run locks.
- `create.ts` — `createResearchRun(db, opts)`: creates a fresh `ResearchRun` in the database, defaulting its status to `PENDING`.
- `runner.ts` — `OnDemandResearchRunner`: coordinates step planning, budget check wall-clock loops (with 5-minute / 10% synthesis buffer), hardware safety checking, execution of step checkpoints (skipping completed ones), and compilation of final or partial reports as markdown files in `data/research/<runId>.md`. The `theme_proposal` run type plans a `theme_proposal_sweep` step: gather recent catalyst and news keyword clusters, call `completeJson` (judge role → deep profile) to label clusters and write rationales, persist a pending `ThemeProposal` row, and compile a proposal report. The `filing_diff` run type (P8) plans one resumable step per watchlist+portfolio symbol (budget-scaled via `maxTickers`): fetch the two most recent 10-K (else 10-Q) primary docs through the shared EDGAR limiter (200KB cap each, injectable `fetchImpl`/`userAgent` for tests), run the PURE `src/monitor/filing-diff` core, `completeJson` (narrator role → fast profile) over ONLY the changed paragraphs, and persist a `FilingEvent` row (kind `filing-diff`, item `diff`, severity = LLM verdict `routine|notable|thesis-relevant`). Per-symbol failures are caught and recorded, never fatal; an invalid LLM response degrades to a deterministic summary with severity `notable`.

## Tests

- `runs.test.ts` — unit tests verifying budget calculations, safety parsers, process/lock reconciliations, and the runner execution loop (graceful budget timeouts, step resume logic, pause/cancel signaling, and the filing_diff/theme_proposal pipelines with fake fetch + `FakeProvider`).
